# Eteya Content Roadmap (v1.0)

Datadriven content-plan för Eteyas blog. Baserad på:
- **Eteyas service-offering** (AI-agenter, process-automation, AI-telefonist)
- **Web-foundation v4.0 §5.1** (topic clusters)
- **Telestore-ranking-mönster** (validerat fungerar i svensk B2B SEO)
- **Buyer journey** (awareness → evaluation → decision)
- **Differentiering**: konkret resultat + svensk SMB-fokus + speed

> **Princip**: Varje artikel har ETT unikt primärt keyword + UNIK search intent.
> Inga artiklar konkurrerar med varandra (no cannibalization).

## Strategisk grund

| Aspekt | Värde |
|---|---|
| Målgrupp | Svenska SMB 10-200 anställda |
| Buyer stages som täcks | Awareness + Evaluation + Decision |
| USP-vinkel | Konkret/mätbart, svenska case, snabb leverans (2-6 veckor) |
| Tone | Expert, kort, koncist, ingen fluff (per Filip Thai voice) |
| Författare default | Filip Thai (Grundare & VD) — hög auktoritet |

## Topical map — 12 artiklar i 3 pillars

```
PILLAR 1 — AI-agenter (5 artiklar)
├── P1   AI-agenter pillar (Awareness)
├── C1.1 AI-agent kostnad (Evaluation)
├── C1.2 AI-agent vs chatbot (Evaluation)
├── C1.3 Välja AI-leverantör (Decision)
└── C1.4 Case-studies recap (Decision/social-proof)

PILLAR 2 — Process-automation (4 artiklar)
├── P2   Process-automation pillar (Awareness)
├── C2.1 Make vs n8n (Evaluation)
├── C2.2 5 processer att automatisera (Awareness/listicle)
└── C2.3 Identifiera processer för automation (Evaluation)

PILLAR 3 — AI-telefonist (3 artiklar)
├── P3   AI-telefonist pillar (Awareness)
├── C3.1 AI-telefonist för restauranger (Evaluation/industry)
└── C3.2 Implementera AI-telefonist på 2 veckor (Decision/how-to)
```

Total: **12 artiklar × ~1500 ord avg = ~18 000 ord content** spritt över 3-6 månader.

## Föreslagen writing order

**Logik**: Anchor-first (pillar 1) → snabba commercial wins (kostnad, vs chatbot) → social proof → expand till andra pillars.

| Vecka | Artikel | Varför just nu |
|---|---|---|
| **1** | P1 (AI-agenter pillar) | Anchor — alla cluster-artiklar länkar tillbaka till denna |
| **2** | C1.4 (5 case-studies) | Snabb att skriva (repurposar existing case content), hög decision-stage konvertering |
| **2** | C1.1 (kostnad) | High commercial intent — Google rewards på pricing-content |
| **3** | C1.2 (vs chatbot) | Common confusion → educational value, fångar broader audience |
| **3** | P2 (Process-automation pillar) | Andra anchor — börja sprida över pillars |
| **4** | C2.1 (Make vs n8n) | Quick win — mycket sökt comparison, vi har 30+ impl-erfarenhet |
| **4** | C2.2 (5 processer) | Listicle = lätt CTR i SERP |
| **5** | C1.3 (välja leverantör) | Late-stage decision content |
| **5** | P3 (AI-telefonist pillar) | Tredje anchor |
| **6** | C3.1 (restauranger) | Sannergården gör content writing-self |
| **6** | C2.3 (identifiera processer) | Operational depth |
| **7+** | C3.2 (implementation 2 veckor) | Last decision-stage content |

**Tempo**: 2 artiklar/vecka = 6 veckor till hela roadmappen är publicerad.

## Iteration efter publicering

**När 5+ artiklar är publicerade**:
- Setup Google Search Console
- Mät: vilka artiklar drar mest trafik? Vilka rankar för oväntade keywords?
- Refine roadmap baserat på riktig data
- Lägg till cluster-artiklar där det visar volym

**När 10+ artiklar publicerade**:
- Identifiera "rising stars" (artiklar som klättrar)
- Förstärk med fler internal links
- Skriv djupare uppföljnings-content om high-traffic topics

## Files att uppdatera per ny artikel

För varje ny artikel:
1. `content/blog/sv/[slug].mdx` — innehållet
2. `public/images/blog/[slug]-hero.webp` — generated via scripts/generate-hero.py
3. Eventuellt: BLOG_AUTHORING.md "Befintliga tags" om nya tags läggs till

Resten är automatiskt:
- Sitemap.ts → auto-listar via getAllPosts()
- Schema.org → genereras via createArticleSchema()
- llms.txt → auto-listar dynamiskt
- RSS → auto-uppdateras
